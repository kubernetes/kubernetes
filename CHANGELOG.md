# Kubernetes Changelog Index

This file serves as a central index to the per‑release changelog documents for the Kubernetes repository. Each released version of Kubernetes has a dedicated changelog file that records API changes, feature gate updates, bug fixes, and other notable modifications.

## How to use this index
- **Reading a changelog** – Click on a link below to open the full changelog for that particular release.
- **Structure of a changelog** – The changelog files follow the standard Kubernetes format: sections for API changes, feature gates, bug fixes, deprecations, and other important notes. They are organized by component (e.g., `kube-apiserver`, `kubelet`, `kubectl`, `cloud-controller-manager`).
- **Finding a specific change** – Use your browser’s search (Ctrl / Cmd + F) within a release file, or search the repository for keywords.

## Adding a new release entry
When a new Kubernetes release is cut, follow these steps to keep the index up‑to‑date:
1. **Create the changelog file** – Add a new file named `CHANGELOG-<major>.<minor>.md` (e.g., `CHANGELOG-1.37.md`) at the repository root.
2. **Populate the file** – Use the existing changelog files as a template. Include sections for API changes, feature gates, bug fixes, etc.
3. **Update the index** – Insert a link to the newly created file at the top of this list (newest releases first) so that the index remains in reverse‑chronological order.
4. **Commit and PR** – Commit the new changelog file and the updated `CHANGELOG.md` in the same pull request.
