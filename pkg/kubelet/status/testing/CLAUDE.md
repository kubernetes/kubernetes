# Package: testing

## Purpose
The `testing` package provides mock implementations and fake types for testing the status package interfaces.

## Key Types/Structs

- **FakePodDeletionSafetyProvider**: Fake implementation of `PodDeletionSafetyProvider` interface. Has configurable `HasRunning` field to control `PodCouldHaveRunningContainers()` return value.
- **MockPodStatusProvider**: Auto-generated mock (via mockery) for the `PodStatusProvider` interface. Provides mock implementation of `GetPodStatus()`.

## Key Functions

- **NewMockPodStatusProvider**: Creates a new MockPodStatusProvider with test cleanup registration.
- **PodCouldHaveRunningContainers**: Fake method that returns the configured `HasRunning` value.
- **GetPodStatus**: Mock method for retrieving pod status by UID.

## Design Notes

- MockPodStatusProvider is generated by mockery - do not edit manually.
- Uses testify/mock for setting expectations and verifying calls.
- Supports fluent API via Expecter types for setting up mock expectations.
