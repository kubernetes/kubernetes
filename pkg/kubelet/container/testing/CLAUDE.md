# Package: testing

Testing utilities and mock implementations for the kubelet container runtime interfaces.

## Key Types/Structs

- **FakeRuntime**: Mock implementation of the `kubecontainer.Runtime` interface for testing pod and container operations. Tracks called functions, started/killed pods and containers, and supports configurable errors and image pull blocking.
- **FakeStreamingRuntime**: Extends FakeRuntime with streaming capabilities (exec, attach, port-forward).
- **FakeRuntimeCache**: Simple cache wrapper around a runtime for testing.
- **FakeRuntimeHelper**: Mock implementation of RuntimeHelper interface providing DNS, hostname, and container options.
- **FakeOS**: Mocks OS-level operations (Stat, ReadDir, MkdirAll, Symlink, etc.) to avoid filesystem side effects.
- **fakeCache**: Simple Cache implementation delegating to runtime.
- **FakeReadyProvider**: Always-ready sources provider for testing.
- **MockRuntime / MockRuntimeCache**: Auto-generated testify mocks with full expectation support.

## Key Functions

- `NewFakeRuntimeCache()`: Creates a fake runtime cache from a pods getter
- `NewFakeCache()`: Creates a fake pod status cache
- `NewFakeReadyProvider()`: Creates an always-ready provider
- `NewMockRuntime() / NewMockRuntimeCache()`: Create testify mocks with cleanup

## Design Notes

- FakeRuntime records all function calls in `CalledFunctions` slice for assertion
- Supports blocking image pulls with `BlockImagePulls` and `UnblockImagePulls()` for latency testing
- FakeOS allows injecting custom behavior via function fields (StatFn, ReadDirFn, etc.)
- MockRuntime generated by mockery provides fluent expectation API
